## How I Approach Designing Software ##

* New Design Skills Developed
Developed stronger spatial reasoning through object placement and hierarchical transformations.
Learned to design scenes using relative coordinate systems and anchor points for consistent scaling and positioning.
Applied material and lighting considerations early in the design phase rather than treating them as visual afterthoughts.
Practiced thinking about user interaction and navigability while structuring scene layout.

* Design Process Followed
Began with a reference concept to ground the scene visually.
Broke complex objects into simpler geometric primitives.
Designed object hierarchy using relative positioning to ensure consistent transformations.
Iteratively adjusted lighting, materials, and camera placement to improve realism and composition.
Refined textures and object proportions after testing visual output in real time.

* Applying These Tactics in Future Work
Hierarchical modeling and anchor-based positioning can be applied to UI systems, simulations, and modular software architecture.
Early visual prototyping can improve design decisions before deeper optimization work begins.
Structured layout planning reduces downstream debugging and rework in complex systems.

## How I Approach Developing Programs ##

* Development Strategies Used
Transitioned from static geometry to reusable object classes (Brick, Circle, etc.).
Implemented modular collision detection and resolution logic to separate concerns.
Used incremental testing to validate lighting, transformations, and physics behaviors individually.
Focused on clarity and readability in function structure to simplify debugging.

* Role of Iteration

Frequently tested small changes before layering new features.
Refined object proportions and lighting multiple times to improve realism.
Revisited earlier design decisions as understanding improved.

* Evolution of My Development Approach
Early milestones focused on “getting it to render.”
Later milestones moved on to realism, organization, and modular structure.
Moved from experimentation to intentional implementation with clear design goals.
Became more comfortable debugging transformation matrices and lighting calculations systematically.
